    余先生 5/29/2020

    # 封装：

    @BeanProperty给属性赋值，自动生成getter、setter方法

    # 继承：

    语法class 子类名 extends 父类名 { 类体 }

    继承说明

    1、子类继承父类的属性和方法

    2、scala 是单继承

    3、继承的调用顺序：父类构造器->【与之相关的】父类辅助构造器(可选)->子类构造器->【与之相关的】子类辅助构造器(可选)

    4、Scala中属性和方法都是动态绑定，而Java中只有方法为动态绑定。换句话说，就是new的对象是谁，就调用谁的属性和方法，没有Java多态的概念， 举个例子：var stuff Stuff = new Stuff(),即使stuff继承了Person父类，重写了方法和属性， 那么依然使用的是stuff的属性和方法。而Java在这种场景中，使用到的是父类Person的属性，子类的方法，因为Java中只有方法为动态绑定。

    # 抽象：

    # 1、抽象类：

    基本语法

    abstract class Person{
    val name:String //一个属性没有初始化，就是抽象属性
    def pStyle(address:String,uuid:Long):String  //只声明而没有实现的方法，就是抽象方法
    def pMakeMoney(name:String,gtime:String):Long={//普通方法
        println(s"${gtime}$name在北京赚了1W元。")
     } 
}

    注意事项说明

    （1）如果父类为抽象类，那么子类需要将抽象的属性和方法实现，否则子类也需声明为抽象类.

    （2）重写非抽象方法需要用 override 修饰，重写抽象方法则可以不加 override(一般都加override).

    （3）在继承的子类内部中,调用父类的方法使用 super 关键字,但是，super关键字只能调用父类的方法，不能用super调用父类的属性来使用.

    （4）子类对父类属性进行实现：

              父类抽象属性var修饰

              父类非抽象属性只支持val修饰

              因为 var 修饰的为可变变量，子类继承之后就可以直接使用，没有必要重写

    匿名类

    abstract class Person {
 val name: String
 def hello(): Unit
}

object Test {
 def main(args: Array[String]): Unit = {
 val person = new Person {
 override val name: String = "客户群体"
 override def hello(): Unit = println("欢迎来到JHSH!")
 }
 }
}

    # 2、单例对象(主要用于保存静态变量)：

    单例对象也叫伴生对象，伴生对象的名称应该和伴生类名一致,也就是**class类与object相同类名 **

    目的：为了能够和Java语言交互,所有"静态"内容都可以放置在类的伴生对象，单例对象中的属性和方法都可以通过伴生对象名（类名）直接调用访问。

    单例对象注意与包对象进行区分。

    半生类对象的用法说明：（apply方法,其实就是通过工厂模式创建实例化对象）

    （1）通过伴生对象的 apply 方法，实现不使用 new 方法创建对象。

    （2）如果想让主构造器变成私有的，可以在()之前加上 private。

    class Person private(name: String)
    1

    （3）apply 方法可以重载。

    def apply():Person={new Person()} //无参构造器
def apply(name:String):Person={new Person(name)} //但主构造器变成私有的，就可以使用此方法创建有参构造器
    1
    2

    （4）Scala 中 obj(arg)的语句实际是在调用该对象的 apply 方法，即 obj.apply(arg)。用以统一面向对象编程和函数式编程的风格。

    （5）当使用 new 关键字构建对象时，调用的其实是类的构造方法，当直接使用类名构建对象时，调用的其实时伴生对象的 apply 方法。

    //案例
object Test {
 def main(args: Array[String]): Unit = {
 //（1）通过伴生对象的 apply 方法，实现不使用 new 关键字创建对象。
 val p1 = Person()
 println("p1.name=" + p1.name)
 val p2 = Person("bobo")
 println("p2.name=" + p2.name)
 }
}
//（2）如果想让主构造器变成私有的，可以在()之前加上 private
class Person private(cName: String) {
 var name: String = cName
}
object Person {
 def apply(): Person = {
 println("apply 空参被调用")
 new Person("xx")
 }
 def apply(name: String): Person = {
 println("apply 有参被调用")
 new Person(name)
}
}

    # 3、特质：

    1、特质介绍：

    特质=抽象类+接口，在trait中即可以有抽象属性和方法，也可以有具体的属性和方法，另外，所有的 Java 接口都可以当做 Scala 特质使用

    （1）基本语法：

    trait 特质名 {
trait 主体
}
    1
    2
    3

    （2）类继承用法：

    没有父类：class 类名 extends 特质 1 with 特质 2 with 特质 3 ...无父类使用extends

    有父类：class 类名 extends 父类 with 特质 1 with 特质 2 with 特质 3 ...有父类使用with

    （3）所有的 Java 接口都可以当做 Scala 特质使用

    class Teacher extends PersonTrait with java.io.Serializable{}
    1

    （4）动态混入特质,目的：可灵活的扩展类的功能

    //动态混入：可灵活的扩展类的功能
 val t2 = new Teacher with SexTrait {
 override var sex: String = "男"
 }
 //调用混入 trait 的属性
 println(t2.sex)

    2、特质叠加：

    假设场景：class 类名 extends 父类（特质1） with 特质2 ....

    （1）特质之间没关系：但特质1与特质2有方法相同，则继承的子类中override重写即可

    （2）特质之间有关系：特质1与特质2都继承特质3且super调用特质3的方法，方法都相同，则叠加结果会按顺序输出：

    主特质1(父类)的println方法>特质2的println方法>特质3的println方法>......>自己类的println方法>动态混入特质的println方法

    //特质之间有关系案例：
trait Ball {
 def describe(): String = {
 "ball"
 }
}
trait Color extends Ball {
 override def describe(): String = {
 "blue-" + super.describe()
 }
}
trait Category extends Ball {
 override def describe(): String = {
 "foot-" + super.describe()
 }
}
class MyBall extends Category with Color {
 override def describe(): String = {
 "my ball is a " + super.describe()
 }
}
object TestTrait {
 def main(args: Array[String]): Unit = {
 println(new MyBall().describe())
 }
}
//输出结果:my ball is a blue-foot-ball

    特质和抽象类的区别

    1.优先使用特质。一个类扩展多个特质是很方便的，但却只能扩展一个抽象类。

    2.如果你需要构造函数参数，使用抽象类。因为抽象类可以定义带参数的构造函数，而特质不行（有无参构造）。
